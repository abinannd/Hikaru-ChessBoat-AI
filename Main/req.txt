Phase 9 – Step 2: Undo / Redo System

Objective

Implement Undo and Redo functionality for both Human vs Human and Human vs AI games.

Reuse the existing python-chess backend.

Do NOT implement custom board state storage.

────────────────────────────────────────────

IMPORTANT

Reuse:

• python-chess.Board()
• ChessBoard.load_position()
• Move History Panel
• Existing rendering pipeline

Do NOT modify:

• ChessInference
• MovePrediction
• board_encoder.py
• move_encoder.py
• chess_model.py
• inference.py
• train.py
• evaluate.py

────────────────────────────────────────────

Undo

Add an "Undo" button.

When pressed:

• If no moves exist:
  Do nothing.

• Human vs Human:
  Undo one move.

• Human vs AI:
  Undo both the human move and the AI response together, restoring the position before the human's turn.

Use:

board.pop()

Never reconstruct the board manually.

────────────────────────────────────────────

Redo

Add a "Redo" button.

Maintain a redo stack.

When a move is undone:

• Push it onto the redo stack.

When Redo is pressed:

• Reapply the move(s) using python-chess.

If the user makes a new move after Undo:

• Clear the redo stack.

────────────────────────────────────────────

Move History

Synchronize the move history panel.

Undo removes the latest move(s).

Redo restores them.

Move numbering and SAN notation must remain correct.

────────────────────────────────────────────

GUI

Add two buttons:

Undo

Redo

Place them beside or below the existing New Game button.

Buttons should disable automatically when no action is available.

────────────────────────────────────────────

Architecture

Create dedicated methods.

Examples:

undo_move()

redo_move()

can_undo()

can_redo()

clear_redo_stack()

Avoid embedding all logic into button callbacks.

────────────────────────────────────────────

Future Compatibility

Design the implementation so it can later support:

• PGN replay

• Engine analysis

• AI self-play

• Save/Load games

without redesign.

────────────────────────────────────────────

Verification

Verify:

✓ Undo works in Human vs Human

✓ Undo works in Human vs AI

✓ Redo restores moves

✓ Move history updates correctly

✓ Board redraws correctly

✓ New move clears redo stack

✓ Buttons enable/disable correctly

✓ AI remains synchronized

✓ No duplicated board state
