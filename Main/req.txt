Phase 9 – Step 4: Promotion Dialog

Objective

Replace the temporary automatic Queen promotion with a user-selectable promotion dialog.

Only the human player should choose a promotion piece.

The AI may continue using automatic Queen promotion.

Reuse the existing move execution pipeline.

────────────────────────────────────────────

IMPORTANT

Reuse:

• python-chess.Board()
• ChessBoard.load_position()
• Existing move execution
• Existing AI integration

Do NOT modify:

• ChessInference
• MovePrediction
• inference.py
• chess_model.py
• board_encoder.py
• move_encoder.py
• train.py
• evaluate.py

────────────────────────────────────────────

Human Promotion

When a human pawn reaches the promotion rank:

Do NOT immediately execute the move.

Instead:

Display a modal popup dialog.

Allow the player to choose:

• Queen

• Rook

• Bishop

• Knight

Use Kivy Popup or ModalView.

────────────────────────────────────────────

Dialog

Display four selectable piece options.

The dialog should:

• Block interaction with the board.

• Close after a selection.

• Cancel safely if dismissed.

────────────────────────────────────────────

Move Execution

Once a piece is selected:

Construct:

chess.Move(
    from_square,
    to_square,
    promotion=<selected piece>
)

Validate the move.

Push to python-chess.

Refresh using:

load_position(board)

Update:

• Move history

• Status

• Undo stack

• Redo stack

Reuse the existing move execution pipeline.

────────────────────────────────────────────

AI Promotion

Keep the existing behavior.

The AI should continue promoting to Queen automatically.

Do NOT change the inference engine.

────────────────────────────────────────────

Architecture

Create dedicated methods.

Examples:

show_promotion_dialog()

complete_promotion()

cancel_promotion()

Avoid embedding popup logic inside move execution.
